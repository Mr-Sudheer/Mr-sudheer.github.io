# Enumeration

After loading "`http://IP_address/`" in the browser, we land on Home page.

![Home page](/assets/img/Corridor/Corridor-home-page.png)

There are 13 doors in the Corridor and each leads to different pages. Click any of the doors and observe the hash in the URL. All the URLs are given below.

```text

```

Each string contains 32 hexadecimal characters, which strongly resembles md5 hash. I used this [tool](
https://10015.io/tools/md5-encrypt-decrypt) to find original values of the hashes.

After computing the hashes, I found that each hash represents a number. Numbers from 1 through 13. Now I know that adding the hash value of a number to the URL gives me a hint to find the flag. Then I considered trying the values outside of these numbers range. Hashes of next consecutive numbers 14, 15 and 16 returned invalid pages but trying out the hash of '0' returned the flag.

# Conclusion

This room demonstrated how predictable object identifiers and insufficient authorization checks can lead to IDOR vulnerabilities. It also tells us the importance of knowledge on common hash formats such as MD5, SHA, etc I fied is crucial.

# Lessons Learned

1. Learned how **IDOR (Insecure Direct Object Reference)** vulnerabilities can expose unauthorized resources through predictable object identifiers.

2. Learned how **MD5 hashing** works and how weak hashes can be identified or cracked using online databases and wordlists.

3. Learned the importance of validating user authorization on the server side instead of relying solely on **client-controlled identifiers.**
